# レビューチェックリスト

学生リポの更新レビュー時に確認する観点。リポの種類（API 仕様・DB・3D・OCR 等）に応じて該当項目のみ適用する。

## 共通

- [ ] README に研究目的・成果物一覧・再現手順があるか
- [ ] 仕様書と実装（schema / コード）の矛盾がないか
- [ ] 空ファイル・0 バイト markdown・リンク切れがないか
- [ ] 実行時生成物（`.pid`, `.db`, `node_modules/`, `.venv/`）がコミットされていないか
- [ ] 誤字・表記ゆれ（見出し番号欠落、型名の不一致）がないか

## API / ドキュメント系

- [ ] エンドポイントごとに正式仕様書（リクエスト・レスポンス・エラーコード）があるか
- [ ] `student_id` 等の型・意味が全ドキュメントで統一されているか
- [ ] 変更案ドキュメントと正本の役割が分かれているか
- [ ] wordbook / 用語集リンクが dead link になっていないか
- [ ] 大量削除・移動時にインデックス README があるか

## DB 系

- [ ] 仕様書のテーブル数と `schema.sql` の CREATE TABLE 数が一致するか
- [ ] カラム名・NOT NULL 制約が仕様と schema で一致するか
- [ ] `PRAGMA foreign_keys` の有効化
- [ ] 仕様書が複数版（旧・修正版）ある場合、正本がどれか明確か

## 3D / LEGO 系

- [ ] SCAD/STL 等の成果物がリポに含まれるか
- [ ] README のテーマとリポ名・内容が一致するか
- [ ] ファイルエンコーディング（UTF-8 推奨）

## OCR / データセット系

- [ ] `.gitignore` で venv・大容量生成物を除外しているか
- [ ] データセット追加時に README / log へ記録があるか

## Issue 起票の優先度

1. 実装不能・誤解を招く仕様矛盾
2. 成果物欠如・リンク切れ
3. 体裁・誤字（単独でも可だが既存 issue にまとめられるならコメント）
