# Issue 起票の判断

## 新規 issue にする

- 既存 open issue に該当論点がない
- 論点が独立しており、受け入れ条件を明確に書ける
- 再現手順または矛盾の根拠（ファイルパス・行）を示せる

## 既存 issue にコメントする

- 同一論点の進捗更新（一部解消・未解消の追記）
- コミット SHA やファイルパスが変わっただけで本質は同じ
- 受け入れ条件の一部が満たされた通知

## issue を起票しない

- 軽微な好みの差（必須でない）
- 同一セッションで同内容を重複作成しようとしている
- まだ差分を読んでいない段階での推測ベース指摘

## タイトル規約（推奨）

```
[doc]  ドキュメント・仕様の問題
[chore] リポ管理（gitignore、pid 等）
[bug]  実装の不具合（再現可能なとき）
```

## 本文に含める項目

1. 概要（1〜3 文）
2. 問題点（表・箇条書き可）
3. 期待する動作
4. 受け入れ条件（チェックリスト）
5. 環境（ブランチ、`git rev-parse --short HEAD`）
6. 関連 issue（`#N` 参照。`Closes #` は使わない）

## 学生コメントが付いた場合

指導者コメント**より後**の学生コメントは、[issue-student-response.md](issue-student-response.md) に従い検証・返信する。新規 issue は原則起票しない。
